# WEBPAGE KNOWLEDGE BASE

## OVERVIEW
`webpage/` is the Docusaurus-based docs site for neko.m1k1o.net, with source docs, generated config/API docs, and versioned output snapshots.

## STRUCTURE
```text
webpage/
├── docs/              # Source documentation
├── src/               # Docusaurus components/pages
├── scripts/           # Helpers for generated docs content
├── static/            # Static site assets
└── versioned_*        # Generated/versioned docs outputs
```

## WHERE TO LOOK
| Task | Location | Notes |
|------|----------|-------|
| Site config | `docusaurus.config.ts`, `sidebars.ts` | Navigation and site behavior |
| Source docs | `docs/` | Preferred edit target |
| Custom UI | `src/` | React/Docusaurus components |
| Generated config docs | `src/components/Configuration/`, `docs/configuration`, `docs/migration-from-v2` | `generate.js` writes help JSON from source txt files |
| Generated API docs | `scripts/`, `docs/api` | `gen-api-docs` pipeline owns API page generation |

## CONVENTIONS
- Node >= 18 is required by `package.json` engines.
- `tsconfig.json` is editor-oriented; primary validation still comes from Docusaurus build and explicit `npm run typecheck`.
- Config docs are generated by `src/components/Configuration/generate.js`; API docs use the `gen-api-docs` shell entrypoint.

## ANTI-PATTERNS
- Do not edit `versioned_docs/`, `versioned_sidebars/`, `versions.json`, or generated help JSON as if they were canonical source.
- Do not mix docs-site code changes with client-app assumptions; this is a separate stack from `client/`.
- Do not update API/config presentation without checking whether `generate.js`, source text files, or `gen-api-docs` is the real source of truth.

## COMMANDS
```bash
(cd webpage && npm ci)
(cd webpage && npm run build)
(cd webpage && npm run typecheck)
```

## NOTES
- `docs/` is source content; generated/versioned outputs exist for publication, not primary editing.
